## Beginner questions

<AccordionGroup>
  <Accordion title="What is SPECTER in one sentence?">
    SPECTER is a private receiving system that gives you a fresh stealth address for every payment instead of exposing one long-lived public wallet.
  </Accordion>
  <Accordion title="What is post-quantum in simple words?">
    It means the recipient discovery layer is designed around cryptography that is meant to survive the kinds of attacks people worry about from future quantum computers.

    In SPECTER's current implementation, that layer uses ML-KEM-768.
  </Accordion>
  <Accordion title="What is a stealth address?">
    A stealth address is a one-time destination derived by the sender from your public meta-address.

    You do not publish that address yourself in advance.
  </Accordion>
  <Accordion title="What is a meta-address?">
    It is your public SPECTER receiving profile. In current docs terms, it contains `spending_pk` and `viewing_pk`.
  </Accordion>
  <Accordion title="Do I need to understand cryptography to use the product?">
    No. The product flow is still just generate, share, send, announce, and scan.

    The crypto details matter when you want to understand why the privacy model works.
  </Accordion>
</AccordionGroup>

## Security questions

<AccordionGroup>
  <Accordion title="Is SPECTER fully post-quantum today?">
    Not end to end.

    The receiving and discovery side uses ML-KEM-768, but the wallet-compatible spend path still derives a `secp256k1` key for Ethereum compatibility.
  </Accordion>
  <Accordion title="Why do the docs say receiving is stronger than spending?">
    Because the code uses ML-KEM for the shared secret and recipient discovery path, but `specter/specter-crypto/src/derive.rs` still converts that result into a classical Ethereum-compatible spend key.
  </Accordion>
  <Accordion title="Does the API ever return sensitive material?">
    Yes. `POST /api/v1/stealth/scan` returns `stealth_sk` and `eth_private_key` for discovered payments.

    Treat that response as highly sensitive.
  </Accordion>
  <Accordion title="Can outsiders still see that a transaction happened?">
    Yes. SPECTER improves recipient privacy and unlinkability. It does not make the chain itself private.
  </Accordion>
</AccordionGroup>

## Product questions

<AccordionGroup>
  <Accordion title="Where is the live backend?">
    The public API examples in these docs use `https://backend.specterpq.com`.
  </Accordion>
  <Accordion title="What is Yellow in this project?">
    Yellow is the main use case that connects stealth discovery to private channel coordination and trading flows.
  </Accordion>
  <Accordion title="Is Yellow fully settled by this backend?">
    No. The current backend exposes the integration surface, but close behavior still includes placeholder-style fields and does not submit final L1 settlement itself.
  </Accordion>
