## Backend health

```bash
curl -s http://localhost:3001/health | jq .
```

Expected keys:

- `status = "ok"`
- `version`
- `uptime_seconds`
- `announcements_count`
- `use_testnet`

## Registry check

```bash
curl -s http://localhost:3001/api/v1/registry/stats | jq .
```

## CLI check

```bash
cd SPECTER/specter
cargo run --bin specter -- --help
```

You should see subcommands: `generate`, `resolve`, `create`, `scan`, `serve`, `bench`.

## Operational checklist

- Required env vars set (`PINATA_GATEWAY_URL`, `PINATA_GATEWAY_TOKEN`)
- API responds on `/health`
- Key generation endpoint succeeds
- A create/publish/scan loop works end to end
