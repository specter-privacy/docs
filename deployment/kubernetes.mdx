---
title: "Kubernetes deployment"
description: "Run SPECTER API on Kubernetes using the provided container"
---

SPECTER does not ship Helm charts, but you can deploy the container directly.

## Minimal manifest

```yaml
apiVersion: apps/v1
kind: Deployment
metadata:
  name: specter-api
spec:
  replicas: 2
  selector:
    matchLabels:
      app: specter-api
  template:
    metadata:
      labels:
        app: specter-api
    spec:
      containers:
        - name: specter-api
          image: specter-api:local
          ports:
            - containerPort: 3001
          envFrom:
            - secretRef:
                name: specter-env
---
apiVersion: v1
kind: Service
metadata:
  name: specter-api
spec:
  selector:
    app: specter-api
  ports:
    - port: 80
      targetPort: 3001
```

## Notes

- Use Secrets for sensitive env vars.
- Add ingress auth/rate limiting at the edge.
- Persist registry externally if you need durable announcements.
