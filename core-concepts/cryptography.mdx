## ML-KEM-768 constants

| Parameter | Size |
| --- | --- |
| Public key (`spending_pk` or `viewing_pk`) | 1184 bytes |
| Secret key (`spending_sk` or `viewing_sk`) | 2400 bytes |
| Ciphertext (`ephemeral_key`) | 1088 bytes |
| Shared secret | 32 bytes |

![ML-KEM-768 diagram](/images/specter/ml-kem-768-diagram.png)

## Core operations

1. `generate_keypair()` creates ML-KEM-768 keypairs via `ml-kem`.
2. `encapsulate(public_key)` returns ciphertext + shared secret.
3. `decapsulate(ciphertext, secret_key)` recovers the shared secret.
4. `compute_view_tag(shared_secret)` computes a 1-byte filter value.

## View tag efficiency

SPECTER uses a 1-byte view tag space (`256` values):

- False-positive rate: `1 / 256` (~`0.39%`)
- Expected filtering efficiency: ~`99.6%`

## Stealth derivation notes

- Shared secret + recipient key material derive one-time stealth key material.
- Ethereum and Sui compatibility paths use deterministic seed expansion and chain-specific address derivation.
- Domain-separated SHAKE256 labels avoid cross-operation collisions.
